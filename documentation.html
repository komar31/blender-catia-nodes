CATIA Nodes

v0.17.0

Système de nœuds custom façon CATIA pour Blender — construction d'entités géométriques (points, lignes, plans, cercles, courbes, projections, intersections) dans l'éditeur de nœuds natif, indépendant de Geometry Nodes.

Blender minimum 4.2 Type Extension (blender_manifest.toml) Licence GPL-3.0-or-later Éditeur CatiaNodeTree

Vue d'ensemble

Deux niveaux de géométrie

L'addon distingue systématiquement deux rôles : la géométrie de construction (repères, références) et la géométrie de résultat.

  • Wireframe — Point, Line, Plane, Circle, Polyline, Curve, Projection, Intersection, Parallel Curve : jamais rendues (hide_render), affichées en fil (ou en Empty pour un résultat ponctuel) avec un modifier Skin pour rester lisibles, nommées comme le nœud qui les a créées.
  • Résultat — Extrude : le seul nœud qui produit un solide destiné à être rendu.

Chaque nœud réévalue immédiatement à chaque changement (slider, mode, câblage) — aucun bouton à cliquer. Le panneau latéral garde un bouton « Réévaluer tout » en filet de sécurité.

Installation

Deux méthodes

  • Extension (recommandé) — glisser-déposer le dossier zippé sur la fenêtre Blender, ou Preferences > Get Extensions > Install from Disk. Fonctionne sur n'importe quelle machine sans configuration.
  • Manuelle (legacy) — copier le dossier blender_catia_nodes/ dans le répertoire scripts/addons/ du profil Blender, puis l'activer dans Preferences > Add-ons.

Prise en main

Ouvrir l'éditeur CATIA Nodes

  • Basculer une zone en Node Editor, puis choisir CATIA Nodes dans le sélecteur de type d'arbre.
  • Shift+A ouvre le menu Add : les nœuds de construction sous Wireframe, Join et Extrude en entrées directes, Parameters et Math en sous-menus.
  • Le panneau N > CATIA donne accès au bouton « Réévaluer tout ».

Architecture

  • core/engine.py — moteur DAG générique (GNode, EvalContext), indépendant de bpy, avec cache et propagation dirty.
  • core/geometry.py — dataclasses analytiques (GPoint, GLine, GPlane, GCircle, GProfile) et algèbre vectorielle fermée ; l'objet Blender n'est jamais relu comme source de données. Inclut une conversion Catmull-Rom → Bézier pour le nœud Curve (interpolation exacte par les points, échantillonnée en profil).
  • blender_adapter/ — matérialisation via bmesh et bpy.data exclusivement, jamais bpy.ops (trop lent pour un usage interactif). Peut aussi reconvertir un objet Empty ↔ Mesh en place lorsqu'un nœud change de type de résultat selon ce qui est connecté (Intersection, Projection).
  • Chaque changement (slider, mode, câblage) réévalue tout l'arbre ; le cache par nœud garantit que seule la portion réellement modifiée recalcule.
  • Les objets Blender sont ré-adoptés par nom après rechargement d'un fichier — aucun doublon créé.
  • Certains sockets (Extrude.profile, Intersection.element1/2, Projection.element) acceptent plusieurs types réels : Blender ne bloque pas les connexions entre types de sockets custom différents, evaluate() fait le dispatch via isinstance.

Honnêteté

Limitations connues

Aucun nœud « Vector » littéral autonome n'existe encore : seuls Plane.vector et Line.vector fournissent des vecteurs réutilisables dans le graphe.

Le socket Extrude.profile reste coloré PROFILE — y brancher une Line, un Circle, une Polyline ou une Curve relie visuellement des couleurs différentes. Ça fonctionne, ce n'est pas homogène.

Circle ne peut pas encore être branché dans Join (seul Line l'est actuellement) — utiliser Polyline ou Curve pour construire un profil directement à partir de points.

Circle (GCircle) n'est pas encore accepté par Projection ni par Intersection — seuls Point, Line et les profils (Join/Polyline/Curve) le sont.

Curve n'implémente pas une vraie courbure analytique (contrairement à GLine/GCircle/GPlane) : le résultat est directement un profil échantillonné, comme Polyline — seule la façon de calculer ses sommets diffère.
